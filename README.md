# GSNet Fine-tuning Reproduction

[中文](README_CN.md) | English

Fine-tuning and evaluation workflow for **Graspness Discovery in Clutters for Fast and Accurate Grasp Detection** (ICCV 2021).

This repository provides a compact reproduction pipeline around the upstream `graspness_unofficial` implementation:

- data preprocessing
- fine-tuning from the official Kinect checkpoint
- periodic validation during training
- full `test_seen` AP evaluation
- RGB grasp visualization for top1 / top5 / top50 predictions

## Requirements

| Component | Version |
| --- | --- |

| PyTorch | 2.11.0+cu128 |
| CUDA toolkit | 12.8 |
| MinkowskiEngine | 0.5.4 |
| Open3D | 0.19.0 |

Required packages/extensions:

- NumPy
- SciPy
- Pillow
- tqdm
- OpenCV
- Open3D
- TensorBoard
- MinkowskiEngine
- pointnet2 CUDA operators
- knn CUDA operator
- graspnetAPI
- grasp_nms

Custom CUDA extensions must be compiled for the target GPU. If you see:

```text
no kernel image is available for execution on the device
```

rebuild `pointnet2` or `knn` with the correct `TORCH_CUDA_ARCH_LIST`.

## Installation

Clone this repository:

```bash
git clone <this-repo>
cd gsnet-finetuning-reproduction
```

Place the upstream GSNet code at:

```text
external/graspness_unofficial
```

The expected layout is:

```text
gsnet-finetuning-reproduction/
  Makefile
  README.md
  external/
    graspness_unofficial/
  gsnet_reproduction/
    assets/report/
    patches/
    scripts/
    tests/
```

Install and compile dependencies following the upstream GSNet requirements, then apply the compatibility patches in:

```text
gsnet_reproduction/patches/
```

## Data and Weights

Download the GraspNet dataset and official Kinect checkpoint. By default, the Makefile expects:

```text
data/graspnet
weights/kinect/minkuresunet_kinect.tar
```

You can override paths:

```bash
make train \
  DATASET_ROOT=/path/to/graspnet \
  OFFICIAL_CKPT=/path/to/minkuresunet_kinect.tar
```

## Input and Output Formats

### Dataset Input

The workflow expects a GraspNet-style dataset root:

```text
graspnet/
  scenes/
    scene_0100/
      kinect/
        rgb/
        depth/
        label/
        meta/
        annotations/
        camera_poses.npy
        cam0_wrt_table.npy
  grasp_label/
  collision_label/
  models/
  dex_models/
  graspness/                  # generated by preprocessing
```

For single-frame inference, the input is one RGB-D frame plus metadata:

- RGB image: `rgb/XXXX.png`
- depth image: `depth/XXXX.png`
- segmentation label: `label/XXXX.png`
- camera intrinsics: `meta/XXXX.mat`
- camera/table poses: `camera_poses.npy`, `cam0_wrt_table.npy`

The script converts the frame into model input:

```text
point_clouds: float32 [N, 3], camera-frame XYZ points in meters
coors:        float32 [N, 3], point_clouds / voxel_size
feats:        float32 [N, 3], all-one features
```

Default `N` is `15000`.

### Checkpoint Input

Training and evaluation checkpoints are PyTorch `.tar` files with:

```text
model_state_dict
optimizer_state_dict
epoch
```

`optimizer_state_dict` is required only for resume training.

### Prediction Output

Inference outputs GraspNet-format grasp arrays saved as `.npy`. Each grasp row has 17 values:

```text
[score, width, height, depth, rotation(9), translation(3), object_id]
```

The same prediction can be loaded by `graspnetAPI.graspnet_eval.GraspGroup`.

### Main Output Files

Training outputs:

```text
gsnet_reproduction/outputs/finetune_runs/<RUN_NAME>/
  train_logs/
    <RUN_NAME>_epochXX.tar
    log_train.txt
  tensorboard/
  val_probe/
  best_AP_checkpoint.tar
  best_probe.json
  final_report.json
```

Full evaluation outputs:

```text
gsnet_reproduction/outputs/test_seen_epochXX_full/
  scene_0100/...               # prediction dumps
  scene_eval_test_seen_kinect/  # scene-level AP cache
  ap_kinect.npy
  ap_kinect_summary.json
```

Visualization outputs:

```text
report_pngs/top1_rgb_overlay.png
report_pngs/top5_rgb_overlay.png
report_pngs/top50_rgb_overlay.png
summary.json
```

## Training

Fine-tune from the official Kinect checkpoint:

```bash
make train
```

Default training settings:

| Option | Value |
| --- | --- |
| camera | `kinect` |
| max epoch | `10` |
| batch size | `4` |
| learning rate | `1e-4` |
| num point | `15000` |
| voxel size | `0.005` |
| validation scenes | `100-102` |
| validation frames | `0-31` |

Follow training log:

```bash
make tail-train
```

Open TensorBoard:

```bash
make tensorboard
```

Then open:

```text
http://127.0.0.1:6007
```

## Testing

Full `test_seen` evaluation for the official checkpoint:

```bash
make eval-official
```

Full `test_seen` evaluation for a fine-tuned checkpoint:

```bash
make eval-epoch EPOCH=10
```

Print the saved AP summary:

```bash
make show-result EPOCH=10
```

List all AP summaries:

```bash
make list-results
```

Evaluation is split into two stages:

1. `test.py --infer` generates prediction dumps
2. `eval_dumps_by_scene.py` evaluates AP scene by scene

## Model Weights

Official Kinect weight:

```text
weights/kinect/minkuresunet_kinect.tar
```

Fine-tuned checkpoints are saved under:

```text
gsnet_reproduction/outputs/finetune_runs/<RUN_NAME>/train_logs/
```

The best checkpoint selected by periodic validation is saved as:

```text
gsnet_reproduction/outputs/finetune_runs/<RUN_NAME>/best_AP_checkpoint.tar
```

Large checkpoints and outputs are ignored by git.
